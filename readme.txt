這是一個專為測試團隊設計的高效能 TestRail 案例檢索工具。透過多語言關聯字典與自定義權重演算，幫助測試人員從海量的案例庫中快速定位高品質、高相關性的測試內容。

---

## 🚀 核心功能 (Core Features)

* **🔍 精準交集搜尋 (AND Search)**：支援多關鍵字（以空格分隔）。系統僅會顯示**同時包含所有關鍵字**（或其關聯語意）的案例，大幅提升搜尋結果的收斂速度。
* **🌐 三語聯動技術**：自動連動「英、繁、簡」與功能代碼（Key），即便輸入英文也能搜到對應的中文案例。
* **📈 智能權重排序**：基於目錄路徑、標題命中、步驟完整度進行綜合評分，優質案例優先呈現。
* **✨ 自動格式化**：解析 TestRail 原生 HTML 標籤，將步驟轉化為美化編號區塊，提升閱讀體驗。
* **🚦 視覺化狀態燈號**：透過顏色標籤直觀識別作者狀態：
    * 🟢 **綠燈標籤**：在職成員，代表案例處於穩定維護狀態。
    * 🔴 **紅燈標籤**：離職成員，提醒該案例可能需進行內容校閱或更新。

---

## 📊 排序邏輯對照表 (Relevancy Ranking)

系統會針對每一筆案例進行即時分析，依據下列維度決定最終顯示順序：

| 排序維度 | 規則說明 (Logic) | 優先級 (Priority) |
| :--- | :--- | :--- |
| **1. 精準匹配** | 搜尋關鍵字與案例 ID 完全一致 (如：#123) | 🌟 **最高** |
| **2. 功能路徑** | 關鍵字命中目錄名稱 (Section Path)，優先聚合模組案例 | 👑 **極高** |
| **3. 標題關聯** | 關鍵字出現在案例標題 (Title) 中 | 🔥 **高** |
| **4. 內容檢索** | 關鍵字出現在測試步驟或預期結果中 | 📝 **中** |
| **5. 內容完整度** | **核心指標**：根據 **Step 數量** 排序。步驟越紮實，排名越靠前 | 🚀 **關鍵加成** |
| **6. 維護者權重** | 優先顯示核心維護帳號或標準化帳號之撰寫內容 | ⚖️ **微調優先** |
| **7. 狀態處理** | **負向加權**：內容過少、無步驟或作者已離職，排名自動後置 | ⚠️ **後置處理** |

## 🛠️ 檔案結構 (File Structure)

* `OnlineApp.py`: 主程式碼。處理 UI 渲染、交集搜尋邏輯與 API 數據同步。
* `keywords.py`: 搜尋字典檔。定義三語關聯詞組，支援擴充功能關鍵字。
* `users.py`: 使用者管理檔。定義維護者 ID、姓名、權重以及 **is_active**
