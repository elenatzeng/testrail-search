這是一個專為測試團隊設計的高效能 TestRail 案例檢索工具。透過多語言關聯字典與自定義權重演算，幫助測試人員從海量的 Suite 中快速定位高品質、高相關性的測試案例。

---

## 🚀 核心功能 (Core Features)

* **三語聯動檢索**：自動連動「英、繁、簡」與功能代碼（Key），實現跨語言精準對齊。
* **智能權重排序**：基於目錄路徑、標題、步驟完整度與維護者層級的綜合評分系統。
* **自動格式化**：自動處理 TestRail HTML 標籤，並為測試步驟進行美化編號。
* **視覺化狀態管理**：透過顏色燈號（🟢 綠燈 / 🔴 紅燈）直觀識別案例撰寫者的在職狀態。

---

## 📊 排序邏輯對照表 (Relevancy Ranking)

系統會針對每一筆案例進行即時分析，依據下列維度決定最終顯示順序：

| 排序維度 | 規則說明 (Logic) | 優先級 (Priority) |
| :--- | :--- | :--- |
| **1. 精準匹配** | 搜尋關鍵字與案例 ID 完全一致 (如：#123) | 🌟 **最高** |
| **2. 功能路徑** | 關鍵字命中目錄名稱 (Section Path)，優先聚合模組案例 | 👑 **極高** |
| **3. 標題關聯** | 關鍵字出現在案例標題 (Title) 中，判定為直接相關 | 🔥 **高** |
| **4. 內容檢索** | 關鍵字出現在測試步驟或預期結果中，判定為細節命中 | 📝 **中** |
| **5. 內容完整度** | **核心指標**：根據 **Step 數量** 排序。步驟越紮實，排名越靠前 | 🚀 **關鍵加成** |
| **6. 維護者權重** | 優先顯示核心維護帳號或標準化帳號之撰寫內容 | ⚖️ **微調優先** |
| **7. 待完善處理** | **空值篩選**：偵測到無步驟或內容過少，排序自動往後移 | ⚠️ **後置處理** |

---

## 🛠️ 檔案結構 (File Structure)

* `OnlineApp.py`: 主程式碼。處理 UI 渲染、API 通訊與核心排序邏輯。
* `keywords.py`: 搜尋字典檔。定義三語關聯詞組，支援擴充功能關鍵字。
* `users.py`: 使用者管理檔。定義維護者 ID、姓名、權重以及**在職燈號狀態**。
